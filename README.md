# Nifty SaaS

> The SaaS starter you need to get shit done

> [!CAUTION]
> You are free to use this, but know it is currently in rapid fluctuation.

### What is the purpose of this?

This is just a braindump, but here's ultimately what my goal with this starter is.

Building a SaaS starter kit on Laravel is very how do I say this... Lame.. In a way... Because most of what you need just comes out of the box and the ecosystem has a lot of great packages to add all the extra functionality. So because of that I back burnered my original plans with this repository and starter kit.

But I had an idea while trying to figure out which frontend would be better, should I use [Hybridly](https://hybridly.dev/) & Vue because I want to support my buddy [Enzo](https://github.com/innocenzi), or do I use [Inertia](https://inertiajs.com/) & Vue so that way I can maybe at some point build the frontend in a way that we can offer 2 flavors of frontend, Vue or React depending on which you want.

But that got me thinking even harder about stuff... If I choose Inertia then we have this wild ability to essentially "choose your own adventure" with this whole system thanks to Inertia. We as a collective community (Nifty SaaS users) could build and maintain backends and frontends that can be mixed and matched based on what you want as a user.

Want to use Laravel and React? Go ahead, we've got a Laravel + React boilerplate. Want to use Phoenix and Vue? Go ahead.

Thanks to Inertia and the community formed around that, we could build out the frontend aspect in all 3 flavors, React, Vue and Svelte, then build out the backend features in Laravel, Phoenix, Rails or Django.

No matter which combination you choose, you'll get the same experience as a user, but you're not pigeonholed into another Next.js boilerplate.

I think if we can achieve this, Nifty SaaS could be the SaaS boilerplate to end all SaaS boilerplates.

You can discuss this with me [here](https://github.com/aniftyco/saas-starter/issues/11).
